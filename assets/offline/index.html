⚙️ Satu File untuk Semua

Cukup edit lib/config/app_config.dart — tidak perlu menyentuh file lain

lib/config/app_config.dart 
// ── URL utama WebView ───────────────────────────────────
static const String webUrl  = 'https://domain-anda.com';
static const String appName = 'My App';

// ── Toggle fitur ────────────────────────────────────────
static const bool enablePullToRefresh = true;
static const bool enableFileDownload  = true;
static const bool enableFileUpload    = true;
static const bool enableWebRTC        = true;
static const bool enableGeolocation   = true;
static const bool enableOfflineMode   = true;
static const bool fullScreen          = true;

// ── Warna ikon status bar ───────────────────────────────
static const Brightness statusBarIconBrightness = Brightness.light;

// ── URL Routing ─────────────────────────────────────────
static const List<String> allowedDomains  = ['domain-anda.com'];
static const List<String> externalDomains = ['facebook.com'];
📦 Instalasi

Dari download hingga app berjalan — ikuti langkah berikut

1
Download atau Clone Project

Clone repository atau download ZIP, lalu masuk ke direktori project.

git clone https://github.com/your-repo/flutter-webview.git 📋 copy
cd flutter-webview 📋 copy
2
Install Dependensi

Jalankan perintah berikut untuk mengunduh semua package yang dibutuhkan.

flutter pub get 📋 copy
3
Set URL & Nama Aplikasi

Buka lib/config/app_config.dart dan ubah nilai berikut:

lib/config/app_config.dart 
static const String webUrl  = 'https://domain-anda.com';
static const String appName = 'Nama App Anda';

⚠️ Untuk nama yang tampil di homescreen perangkat, ubah juga:
• android:label di AndroidManifest.xml
• CFBundleDisplayName di ios/Runner/Info.plist

4
Ganti Ikon Aplikasi

Ganti assets/offline/webview.png dengan logo Anda (min 1024×1024 px), lalu generate ulang icon:

dart run flutter_launcher_icons 📋 copy
5
Jalankan Aplikasi

Hubungkan perangkat atau jalankan emulator, lalu:

flutter run 📋 copy
flutter devices 📋 copy — lihat daftar device
6
Build Release

Siap distribusi? Build versi release:

flutter build apk --release 📋 copy — Android APK
flutter build appbundle --release 📋 copy — Google Play
flutter build ios --release 📋 copy — iOS App Store
❓ FAQ

Pertanyaan yang sering ditanyakan seputar penggunaan template ini

Mengapa nama di AppConfig.appName tidak berpengaruh ke nama app di homescreen? ›

AppConfig.appName hanya dipakai di dalam kode Flutter (judul MaterialApp, halaman offline, dll.). Nama yang tampil di launcher / homescreen perangkat diambil dari file native:

  • Android: android:label di AndroidManifest.xml
  • iOS: CFBundleDisplayName & CFBundleName di Info.plist
Konten web tidak tampil saat offline, padahal sudah pernah dibuka? ›

Cache bergantung pada header HTTP server. Pastikan server mengirim header yang mengizinkan caching:

Cache-Control: public, max-age=86400

Atau gunakan offlineSource = OfflineSource.localHtml sebagai fallback yang lebih andal — konten HTML lokal selalu tersedia meski tidak ada cache.

Bagaimana cara mengontrol URL mana yang dibuka di WebView vs browser eksternal? ›

Gunakan dua config di app_config.dart:

  • allowedDomains — allowlist: hanya domain ini yang dibuka di WebView. Kosong = semua boleh masuk.
  • externalDomains — blocklist: domain ini selalu dibuka di browser eksternal.
  • externalSchemes — scheme yang dibuka di app eksternal (tel:, mailto:, whatsapp:, dll.).
WebRTC / kamera tidak bekerja — bagaimana cara memperbaikinya? ›

Pastikan beberapa hal berikut:

  • enableWebRTC = true di app_config.dart
  • Halaman web menggunakan HTTPS — WebRTC memerlukan secure context
  • Izin kamera & mikrofon sudah ada di AndroidManifest.xml dan Info.plist (sudah dikonfigurasi otomatis)
  • Di iOS: pastikan NSCameraUsageDescription dan NSMicrophoneUsageDescription ada di Info.plist
Bagaimana cara mengirim data dari Flutter ke JavaScript (dan sebaliknya)? ›

Flutter → JavaScript: gunakan evaluateJavascript

await controller.evaluateJavascript(source: "window.token = '$token';");

JavaScript → Flutter: tambahkan JavaScript handler di onWebViewCreated

controller.addJavaScriptHandler(handlerName: 'myHandler', callback: (args) { ... });

Di JavaScript: window.flutter_inappwebview.callHandler('myHandler', data);

Ikon aplikasi terpotong atau terlalu besar — bagaimana memperbaikinya? ›

Siapkan file PNG dengan ukuran minimal 1024×1024 px dan tambahkan padding sekitar 10–20% di semua sisi agar ikon tidak terpotong saat di-crop menjadi lingkaran (adaptive icon Android) atau bentuk rounded (iOS).

Setelah mengganti file, generate ulang:

dart run flutter_launcher_icons

Untuk upload ke App Store iOS, tambahkan remove_alpha_ios: true di konfigurasi flutter_launcher_icons di pubspec.yaml.

Bagaimana cara mengaktifkan deep link HTTPS (App Links / Universal Links)? ›

Android (App Links):

  • Uncomment intent-filter HTTPS di AndroidManifest.xml
  • Upload file verifikasi ke https://domain-anda.com/.well-known/assetlinks.json

iOS (Universal Links):

  • Aktifkan capability Associated Domains di Xcode
  • Upload file ke https://domain-anda.com/.well-known/apple-app-site-association
  • Set universalLinkDomain = 'domain-anda.com' di app_config.dart
Status bar terlalu gelap atau terlalu terang — bagaimana mengaturnya? ›

Atur warna ikon status bar (jam, baterai, sinyal) di app_config.dart:

  • Brightness.light → ikon putih — untuk app dengan background gelap
  • Brightness.dark → ikon hitam — untuk app dengan background terang

static const Brightness statusBarIconBrightness = Brightness.light;
